## 4. How To Enter

1. Register on gitlab.devpost.com
2. Request access to GitLab AI Hackathon group via: https://forms.gle/EeCH2WWUewK3eGmVA
3. Complete "Enter a Submission" page

### Project Requirements

- **What to Create**: A working AI agent or flow built on **GitLab Duo Agent Platform** that helps with the **SDLC**. Must perform a specific action or workflow automation (NOT just a text-based chat).
- **Functionality**: Must run consistently on GitLab Duo Agent Platform and function as depicted in video/description.
- **Platform**: Must be published in the GitLab AI Hackathon group: https://gitlab.com/gitlab-ai-hackathon
- **New & Existing**: Must be newly created OR significantly updated after Feb 9, 2026.
- **Third Party**: If integrating third-party SDK/APIs, must be authorized to use them.

### Submission Requirements

1. A Project built with GitLab Duo Agent Platform
2. URL to the Project — all YAML files must be original work; repo must be public; MIT License for original work
3. Text description explaining features and functionality
4. Demo video:
   - Less than 3 minutes (judges not required to watch beyond 3 min)
   - Show the Project functioning on the device
   - **Must clearly show the Project reacting to a trigger and performing an action**
   - Uploaded to YouTube/Vimeo/Facebook Video/Youku (public)
   - No unauthorized third-party trademarks or copyrighted music

### Submission IP Requirements

- Original work: Subject to **MIT License** and GitLab's Developer Certificate of Origin (v1.1)
- Third-party components: Must comply with OSI-approved open source licenses
- Must not violate IP rights of any other person/entity

## 6. Judges & Criteria

### Stage One — Pass/Fail

Does the Project reasonably fit the theme and apply the required APIs/SDKs?

### Stage Two — Equally Weighted Criteria

1. **Technological Implementation**: Quality software development? Leverages GitLab Duo Agent Platform (Tools, Triggers, Context)? Code quality?
2. **Design**: UX well thought out? Easy to install, configure, and interact with? Workflow makes sense?
3. **Potential Impact**: How big of an impact on target community? Solves a real "AI Paradox" bottleneck (reducing manual toil in planning, security, or operations)?
4. **Quality of the Idea**: How creative and unique? Does the concept exist already? How much improvement?

**Tie Breaking**: Highest score in first criterion → second → third → fourth → judges vote.
